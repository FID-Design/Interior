# FID 피드디자인 네이버 SEO 전략 가이드
### 강남 인테리어 업체 · 정적 사이트(github.io) 기준 · 2026년 최신 알고리즘 반영

> 작성 관점: 정적 사이트(github.io)는 네이버에서 **검색 노출의 주력이 될 수 없습니다.** 이 가이드의 핵심 결론은 "사이트 최적화"가 아니라 **블로그 + 스마트플레이스 + 사이트의 3채널 투트랙 운영**입니다. 사이트는 브랜드 전환(견적·상담) 랜딩으로 두고, 네이버 유입은 블로그→플레이스→사이트 순으로 설계합니다.

## 1. 네이버 검색엔진은 어떻게 동작하는가

### 1-1. 한 장 요약

| 단계 | 무슨 일이 일어나는가 |
|---|---|
| 수집(크롤링) | 네이버 로봇 **Yeti**가 `robots.txt`를 지키며 페이지를 긁어온다 |
| 정제·색인 | 본문이 빈약하거나 스팸 의심이면 색인에서 빠진다 (색인 ≠ 노출) |
| 랭킹 | **C-Rank**(출처 신뢰도) + **D.I.A / D.I.A+**(문서·의도 적합도)를 합산해 순위 결정 |
| 노출 | 통합검색이 **스마트블록**으로 의도별로 결과를 쪼개 보여준다 |

### 1-2. C-Rank — "누가 썼는가"(출처 신뢰도 점수)

- 블로그·사이트 같은 **출처(주체)** 단위로 매기는 신뢰도 점수입니다.
- 네이버는 콘텐츠를 **31개 주제**로 분류하고, **한 주제에 얼마나 꾸준하고 전문적으로** 양질의 글을 쌓았는지를 봅니다.
- 핵심 함의: **"인테리어 한 주제만 파는 블로그"가 유리합니다.** 인테리어+맛집+여행을 섞으면 주제가 분산돼 점수가 떨어집니다.

### 1-3. D.I.A / D.I.A+ — "이 문서가 검색의도에 맞는가"(문서 점수)

- **D.I.A(Deep Intent Analysis)**: 제목-본문 적합도, **실제 경험 정보**(직접 시공한 1차 정보), 서론-본론-결론 구조를 평가.
- **D.I.A+**: 검색자의 **구체적 질의 의도**를 문서가 얼마나 반영했는지(반영도)를 평가. 체류시간·스크롤·공감·댓글 같은 **사용자 행동 신호**도 반영.
- 핵심 함의: 분량만 길고 실제 시공 사례·사진이 없는 글은 불리합니다. **시공 전후 사진 + 실제 후기**가 점수의 핵심.

### 1-4. 통합검색 구조 — 스마트블록

- 네이버는 구글처럼 한 줄로 순위를 매기지 않고, **검색의도별로 결과를 잘게 쪼갠 블록**으로 화면을 구성합니다.
- 예: "강남 인테리어"를 치면 `강남 인테리어 비용`, `아파트 인테리어 추천`, `인테리어 시공 후기` 같은 세부 블록과 **플레이스(지도) 탭, 블로그/VIEW 탭**이 따로 노출됩니다.
- **상업 키워드(인테리어)는 플레이스(지도) + 블로그가 화면을 압도하고, "웹사이트" 탭은 하단 보조 노출에 머뭅니다.**

### 1-5. Yeti 크롤러

- User-agent: `Yeti` (`Mozilla/5.0 (compatible; Yeti/1.1; +http://naver.me/spd)`)
- `robots.txt`를 준수합니다. → **Yeti를 막거나 CSS/JS를 차단하면 색인에서 빠집니다.** 정적 사이트는 전체 허용이 정답.
- 색인까지 보통 **2~4주** 소요. 며칠 만에 노출 안 된다고 설정을 계속 바꾸면 오히려 불안정해집니다.

### 1-6. 네이버 vs 구글 — 결정적 차이

| 구분 | 네이버 | 구글 |
|---|---|---|
| 핵심 랭킹 신호 | 자사 생태계(블로그·카페·플레이스) UGC + **사용자 행동**(클릭·체류·전화·길찾기·재방문) | **백링크 권위(PageRank)** + 유용성(HCU) |
| 외부 백링크 가중치 | **낮음** | 높음 |
| 결과 화면 | 스마트블록 큐레이션(탭·블록 분산) | 단일 랭킹 리스트 |
| 정적 사이트 노출 공간 | **매우 좁음**(상업 키워드는 특히) | 상대적으로 넓음 |
| 색인 속도 | 2~4주, 데이터 범위 좁음 | 빠름, 16개월 데이터·URL 검사 풍부 |

> **한 문장 결론:** 네이버에서 "강남 인테리어"는 백링크로 이기는 게임이 아니라, **블로그 전문성(C-Rank)과 플레이스 행동 데이터로 이기는 게임**입니다.

## 2. 인테리어 키워드 상위노출 핵심 전략

### 2-1. 대원칙

- 대표 키워드(`인테리어`, `리모델링`)는 대형 플랫폼·광고가 점유 → **진입 불가.**
- 전략은 **지역(강남) + 공간유형 + 행동(의도)** 조합의 **롱테일로 진입**하는 것.
- 검색량만 보지 말고 **검색량 × 경쟁강도 × 의도**를 함께 봅니다. (네이버 검색광고 키워드도구 + 데이터랩으로 월 1회 점검)

### 2-2. 의도별 키워드 매핑 (한 키워드 = 한 페이지/한 채널)

| 검색의도 | 대표 키워드 예시 | 담당 채널 | 우선순위 |
|---|---|---|---|
| 거래(견적·상담) | `강남 인테리어 견적`, `강남 상가 인테리어 비용`, `강남 사무실 인테리어 상담` | **플레이스 + 파워링크 + 사이트 견적폼** | ★★★ |
| 비교(업체·후기) | `강남 인테리어 업체`, `강남 인테리어 후기`, `강남 아파트 인테리어 추천` | **블로그 + 플레이스 리뷰** | ★★★ |
| 정보(설계·스타일) | `강남 아파트 인테리어 시공사례`, `30평 인테리어`, `모던 카페 인테리어` | **블로그 + 사이트 포트폴리오** | ★★ |

### 2-3. 우선 공략 롱테일 목록 (예시)

- `강남 아파트 인테리어 시공`
- `강남 상가 인테리어 견적`
- `강남구 사무실 인테리어`
- `테헤란로 인테리어 업체`
- `강남 카페 인테리어 사례`
- `강남 골프장/골프아카데미 인테리어`
- `강남 30평 아파트 인테리어 비용`
- `강남 리모델링 후기`

### 2-4. 스마트블록 공략법

- VIEW 단일 최상단 하나만 노리지 말 것. **비용/추천/후기/평수별/스타일별** 의도를 **제목과 소제목(H2/H3)에 분산 배치**해 여러 세부 블록에 동시에 걸리게 합니다.
- 페이지/글당 키워드는 **5~6개 이내.** 기계적 반복(키워드 스터핑)은 스팸 필터를 부릅니다.

## 4. 정적 github.io 사이트의 한계와 현실적 보완책

### 4-1. 구조적 한계

| 한계 | 왜 문제인가 |
|---|---|
| 지도/플레이스 탭 노출 **불가** | 지도 노출은 **스마트플레이스 등록**이 있어야만 가능. 사이트는 웹문서 채널로 별개 |
| C-Rank/D.I.A 로직 **미적용** | 이 로직은 블로그/카페 UGC 평가용. 정적 사이트는 색인만 받고 자연순위 공간이 매우 좁음 |
| **갱신 빈약**(freshness 약함) | 페이지가 적고 자주 안 바뀌면 "지속 갱신 출처" 조건 미충족 |
| 서버측 **301 리다이렉트 불가** | www 유무·http/https 혼재 시 중복 색인 → canonical로만 교정 가능 |
| RSS·사이트맵 **자동갱신 없음** | 새 시공사례 추가 시마다 사이트맵 **수동 갱신 후 재제출** 필요 |
| 기본 `github.io` 도메인 | 브랜드 신뢰·색인에 불리 |

### 4-2. 현실적 보완책 (투트랙)

- **블로그·포스트 병행 (필수):** 네이버 노출의 본진은 블로그. 사이트는 **브랜드 전환 랜딩**으로, 블로그→플레이스→사이트 순서로 유입 설계.
- **커스텀 도메인 (강력 권장):** `fid-design.co.kr` 등 자체 도메인 연결 + HTTPS. **반드시 서치어드바이저 등록 전에 연결**(나중에 바꾸면 색인 자산이 분산).
- **백링크는 "자연 언급"만:** 구글식 유료 대량 백링크는 네이버에서 효과 적고 **어뷰징 역효과**. 대신 협력사 카페·블로그에서의 **자연스러운 언급**, 그리고 **플레이스에 연결된 블로그 리뷰 언급량**(네이버에선 이게 사실상 사이테이션 역할)을 늘립니다.
- **인스타는 보조 채널:** 2025년 7월부터 공개 전문가 계정이 구글·빙엔 색인되나 **네이버 직접 상위노출 경로는 제한적**. 캡션 키워드·위치태그·브랜드 도메인 링크 정도로 보조 활용.
- **수동 운영 루틴화:** 시공사례 추가 = (1) 사이트 개별 페이지 생성 → (2) sitemap.xml `<url>` 블록 추가 → (3) 서치어드바이저 재제출, 을 한 세트로.
